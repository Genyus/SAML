Threat Model Name: Telligent Community SAML Create User Flow

Owner: Dan Bartels

Reviewer:

Contributors:

Description: Describes the SAML Authentication flow of a new User to the community

Assumptions: The community natively supports OAuth tokens to delegate user identity. The SAML claims are persisted into an encrypted cookie and used for the lifetime of the Login Forms Create User Process. (this data is used to create a user, as the site requires the claims to be persisted during multiple postbacks). Users are associated with the SAML identity through a mock OAuth response in temporary storage. ASPNET forms authentication tokens are used to facilitate user identities on the site.
